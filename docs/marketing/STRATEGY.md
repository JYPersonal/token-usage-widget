# Launch & marketing strategy

Goal: get developers who burn Codex / Cursor quotas to **star, clone, and run** the widget in one sitting.

## Core angle (use everywhere)

> Local Windows corner HUD for Codex + Cursor usage — no more tab-hopping billing pages.

Supporting truths (always honest):
- Local-only, MIT, no SaaS account for this app
- Best today: Codex + Cursor; other providers fail closed
- Windows widget; dashboard works anywhere Node runs

## Audience priority

1. **Cursor + Codex daily users** (highest conversion)
2. Multi-CLI / agent harness builders
3. Open-source tool collectors (awesome-lists, HN lurkers)

## Channel plan (first 14 days)

| Day | Channel | Action |
| --- | ------- | ------ |
| 0 | GitHub | Topics, description, Release `v1.0.0`, polished README |
| 0 | X | Hook post + image; thread link in reply |
| 1 | Reddit | One sub first (`r/cursor` or `r/ChatGPTCoding`); engage every comment |
| 1–2 | Dev.to | Long-form how-to; cross-link GitHub |
| 2 | daily.dev | Squad / post using Dev.to or GitHub URL |
| 3 | HN | **Show HN** once (not same hour as Reddit mega-thread) |
| 5–7 | awesome-* PRs | Cursor tools, Electron, AI tooling lists |
| 7–14 | Follow-ups | Reply GIFs, “how I set defaults”, limitation honesty posts |

## Posting rules that protect trust

- Lead with **pain + screenshot**, not feature laundry lists
- Put the **repo link once**, clearly
- State **Windows widget** upfront so Mac users aren’t angry
- Don’t claim all 8 providers are equally solid
- Don’t buy engagement; reply like a maintainer

## Success metrics (lightweight)

- Stars / forks in first 72h
- Issues that are real usage reports (good signal)
- Clones implied by “it works” comments
- Referral traffic from HN vs Reddit vs X (UTM optional: `?utm_source=hn`)

## Amplifiers (optional)

- 60–90s Loom: install → `setup:defaults` → widget appears
- Offer a one-line blurb others can quote when they share
- Ask 3–5 friends who live in Cursor/Codex to star + try on Day 0
